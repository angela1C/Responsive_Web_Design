Bootstrap4

Build fast, responsive sites with Bootstrap Quickly design and customize responsive mobile-first sites with Bootstrap, the world’s most popular front-end open source toolkit, featuring Sass variables and mixins, responsive grid system, extensive prebuilt components, and powerful JavaScript plugins.

From https://getbootstrap.com
Bootstrap4 is the most popular HTML, CSS, and JavaScript framework for developing responsive, mobile-first websites. This guide is not a comprehensive guide but instead provides an overview of some of the main features of Bootstrap4 based on the documentation at https://getbootstrap.com

Note: There is a v5.0.0-beta1 version.

Bootstrap is a free front-end framework for faster and easier web development.

  • Bootstrap requires the use of the HTML5 doctype.
  • Bootstrap is developed mobile-first. The code is optimised for mobile devices first and components are scaled up as necessary using CSS media queries.
  • Add the responsive viewport meta tag to you <head> to ensure proper rendering and touch zooming for all devices.
  • Bootstrap supports the latest, stable releases of all major browsers and platforms.
  • Bootstrap supports the latest versions of each major platform’s default browsers.
  • You can download Bootstrap or use a CDN. See Getting Started

Layout

This section outlines the components and options for laying out a Bootstrap project including wrapping containers, a powerful grid system, flexible media object and responsive utility classes.

Containers are the most basic layout element in Bootstrap and are required when using it's default grid system. Use containers to contain, pad and center the content within them. Containers can be nested but this is not usually required.

    The 3 Bootstrap4 containers:
  • The default .container is a responsive, fixed-width container. This means that it's max-width changes at each responsive breakpoint
  • The .container-fluid class uses width: 100% at all breakpoints so that the content spans the entire width of the viewport
  • Responsive containers using.container-{breakpoint} allows you to specify width: 100% until the specified breakpoint is reached.

    The following examples are provided at getbootstrap.com
  • <div class="container-sm">100% wide until small breakpoint is reached where it will scale up with md, lg and xl. </div>
  • <div class="container-md">100% wide until medium breakpoint</div>
  • <div class="container-lg">100% wide until large breakpoint</div>
  • <div class="container-xl">100% wide until extra large breakpoint</div>

Bootstrap is developed to be mobile-first. It uses a few media queries to create sensible breakpoints for its layouts and interfaces, the breakpoints are mostly based on the minimum viewport widths. They allow for elements to be scaled up as the viewport changes. Refer to the documentation for more information on this and how it works.

Some Bootstrap components use z-index which is a CSS property that helps control layout by providing a third axis to arrange content. Bootstrap utilizes a default z-index scale in Bootstrap that’s been designed to properly layer navigation, tooltips and popovers, modals, and more. Bootstrap uses a default z-index scale that allows proper layer navigation, tooltips and popovers, modals and more. See here for more details. These values start at an arbitrary number that is high and specific enough to ideally avoid conflicts. It is not recommended to customise these values. 

        $zindex-dropdown:1000 !default;
        $zindex-sticky:1030 !default;
        $zindex-modal-backdrop:1040 !default;
        $zindex-fixed::1050 !default;
        $zindex-popover::1060 !default;
        $zindex-tooltip:1070 !default;

Low single digit numbers such as 1, 2 and 3 overlapping borders and components such as buttons and inputs in input groups, are used to handle for default, hover and active states. On hover/focus/active, a particular element is brought to the forefront with a higher z-index value to show their border over the sibling elements.

Grid System

See Grid system which uses a mobile-first flexbox grid to built all sorts of layouts.

The bootstrap grid system is built with Flexbox and uses a series of containers, rows and layouts to layout and align content. It is fully responsive.

  • Containers are a way to centre and horizontally pad your content on the page.
    • .container for a responsive pixel width
    • .container-fluid for width:100% across all viewport and device sizes

    • Rows are a wrapper for columns.
    • Content must be placed in columns in the grid layout
    • Only columns can be direct children of rows.
    • Each column has a horizontal padding or gutter to control the space between columns. The margin can be removed from rows and the padding from columns using .no-gutters on the .row
    • Without specifying a width, each column will have equal width
    • Column classes indicate the number of columns to use out of a possible 12 per row.
    • Column width is set in percentages to they are fluid and sized relative to their parent element.
    • There are five breakpoints (one for each responsive breakpoint) to make the grid responsive.
    • All breakpoints (extra small), small, medium, large and extra large.
    • Grid breakpoints are based on minimum width media queries. This means that they apply to that one breakpoint and all those above it. For example .col-md-4 would apply to medium, large and extra larget devices but not small and extra small breakpoints.
    • Predefined grid classes such as .col-4 can also be used.
    • Bootstrap uses pxs for grid breakpoints and container widths. This is because the viewport width is in pixels and doesn't change with the font size.(Bootstrap does use ems and rems for defining most sizes).
    • See grid options for more details on how the grid system works across multiple
    • breakpoint specific column classes can be used for easy column sizing without an explicit numbered class. For example .col-sm-6
    • Equal width will be applied to columns within a row if no units are supplied.
    • Use .w-100 to create equal-width columns that span multiple-lines. Insert the .w-100 where you want the columns to break to a new line.
    • You can set one column's width and have the other sibling columns in that row automatically resize around it.
    • To size columns based on the natural width of their content use col-{breakpoint}-auto classes.
    • Use .col and .col* classes for grids that are the same for all devices from smallest to largest.
    • Specify a numbered class if you need a particularly sized column. For example class="col-4
    • Change from a stacked grid system to one that becomes horizontal at the smallest breakpoint(sm using a single set of .col-sm-* classes.
    • Use a combination of different classes for each tier if you don't want columns to stack in some tiers.
    • Gutters can be responsively adjusted by breakpoint-specific padding and negative margin utility classes. See gutters for example in the docs.
    • Remove gutters between columns in predefined classes using .no-gutters
    • Use .row-cols-* to quickly set the number of columns that best render the content and layout. This is set in the parent .row as a shortcut. (Normal .col-* apply to individual columns.)
    • Alternatively use the Sass mixin row-cols().

    • Flexbox utilities are used to vertically and horizontally align columns.
        • Vertical Alignment:
      • align-items-start, align-items-center, align-items-end
      • align-self-start, align-self-center, align-self-end
        Horizontal Alignment:
      • row justify-content-start, row justify-content-center, row justify-content-around, row justify-content-between
    • Column wrapping occurs when more than 12 columns are placed on a single row. Each group of extra columns wraps as a unit to the next line.
    • Use width:100% to wrap columns to a new line
    • Reordering: Use .order- classes to control the visual order of the content. These classes are responsive so the order can be set by breakpoint. See reordering for examples.
    • Columns can be moved to the right using .offset-me-* classes to increase the left margin by * columns.
    • .offset-md-4 moves .col-md-4 over 4 columns.
    • Margin utilities such as .mr-auto can be used to force sibling columns away from each other.
    • To nest content within the default grid, add a new .row and a set of .col-sm-* columns within an existing .com-sm-* column.
    • Nested rows should include a set of rows that add up to 12 or fewer.
    • See Sass mixins section on using Sass variables and minins to create custom, semantic and responsive page layouts.
Content: Reboot

Reboot is a collection of element-specific CSS changes in a single file. Reboot provides an elegant, consistent and simple baseline to build on. It builds upon Normalise providing many HTML elements with styles using only element selectors. Additional styling is done with classes.

    Some guidelines are provided if you want to override elements of Reboot:
  • Update some browser default values to use rems instead of ems for scalable component spacing.
  • Avoid margin-top because vertical margins can collapse with unexpected results.
  • Block elements should use rems for margins for easier scaling across device sizes.
  • Have minimal declarations of font-related properties. Use inherit whenever possible.

  • The html and body elements are updated to provide better page-width defaults.
  • The box-sizing is set globally on every element to border-box to ensure that the declared width of an element is never exceeded due to padding or border.
  • No base font-size is declared on the html byt 16px is assumed.
  • font-size: 1rem is applied on the <body> for easy responsive type-scaling via media queries.
  • The <body> also sets a global font-family,line-height and text-align
  • The <body> has a declared background-color defaulting to #fff.
  • Native Font stack:The default web fonts (Helvetica Neue, Helvetica, and Arial) have been dropped in Bootstrap 4 and replaced with a “native font stack” for optimum text rendering on every device and OS.
  • Headings and paragraphs:All heading elements from <h1> to<h6> and paragraphs <p> are reset to have theirmargin-top removed.
  • Headings have margin-bottom: .5rem added
  • Paragraphs have margin-bottom: 1rem for easy spacing.
  • All lists <ul>, <ol>< and <dl> have their margin-top removed and a margin-bottom:1rem
  • Nested lists have no margin-bottom for a more even appearance.
  • Description lists <dd> have margin-left reset to 0 and margin-bottom: .5rem added. <dt>'s are bolded.
  • Preformatted text: The <pre> is reset to remove it's margin-top and uses rem units for its margin-bottom.
  • Tables: Tables are slightly adjusted to style <caption>s, collapse borders and ensure consistent text-align throughout.
  • Additional changes for borders, paddings etc come with the .table class.

    • Forms: Various form elements are rebooted for simpler base styles:

    • <fieldset> have no borders, padding or margin
    • legends are restyled to be displayed as a heading of sorts.
    • labels are set to display:inline-block to allow margin to be applied.
    • The margin is removed and line-height:inherit set for <input>, <select>, <textarea> and <button>s
    • <textarea> are modified to only be resizable vertically.
    • <button>s and <input> button elements have cursor: pointer when :not(:disabled).
    • See Forms for demo and further details.
  • Reboot includes an enhancement for role="button" to change the default cursor to pointer. This can be added to elements to show that they are interactive elements.
  • The <address> element is updated to reset browser default font-style from italic to normal.
  • The margin is reset to 0 0 1rem for blockquotes to be more consistent with other elements.
  • The <abbr> element has some basic styling to make it stand out from paragraph text.
  • The default cursor on summary is reset to pointer to show that it can be interacted with by clicking on it.

Content: Typography

Bootstrap sets basic global display, typography, and link styles. See textual utility classes for when more control is needed.

Global Settings

  • Bootstrap uses a native font stack that selects the best font-family for each OS and device.
  • The browsers default root font-size of 16px is used.
  • The $font-family-base, $font-size-base and $line-height-base attributes as the typographic base applied to the <body>.
  • Set the global link color using $link-color, apply link underlines on :hover.
  • Use $body-bg to set a background-color on the <body> which is #fff by default.
  • All HTML headings <h1> to <h6> are available as well as .h1 to .h6 classes for when you want to match the font-styling of a headng but cannot use the associated HTML element.
  • Customise headings using utlity classes. For example class="text-muted"
  • Display headings are headings that will stand out more than traditional heading elements. For example using class="display=1"
  • Use .lead to make a paragraph stand out as the lead paragraph. For example: class="lead"

    Inline Text elements: styling for common HTML5 elements

  • Use the <mark> to highlight text.
  • Use <del> to show a text as deleted text.
  • Use <s> to indicate that text should no longer be treated as accurate.
  • Use <ins> to indicate text treated as an addition to the document.
  • Use <u> to render text as underlined.
  • Use <small> to treat text as fineprint.
  • Use <strong> to render text as bold.
  • Use <em> to render text in italics.
  • The .mark and .small classes are also available to apply the same styles as <mark> and <small> while avoiding any possible unwated semantic implications from using the tags.
  • The HTML5 <b> element can be used for higlighting word or phrases without conveying additional importance.
  • The HTML5 <i> tags can also be used for voice and technical terms.

    Images

    Documentation and examples for opting images into responsive behavior (so they never become larger than their parent elements) and add lightweight styles to them—all via classes.

    From https://getbootstrap.com/docs/5.0/content/images/
  • Images in Bootstrap are made responsive with .img-fluid. This applies max-width: 100%; and height: auto; to the image so that it scales with the parent element.
  • As well as border-radius utilities, you can use .img-thumbnail to give an image a rounded 1px border appearance.
  • Images can be aligned with the helper float classes or text alignment classes
  • Centre block level images using the .mx-auto margin. utility class

    • Tables:

    Due to the widespread use of

    elements across third-party widgets like calendars and date pickers, Bootstrap’s tables are opt-in. Add the base class .table to any
    , then extend with our optional modifier classes or custom styles. All table styles are not inherited in Bootstrap, meaning any nested tables can be styled independent from the parent.

    From https://getbootstrap.com/docs/5.0/content/tables/
    • Add the base class .table to any <table>
    • Extend with optional modifier classes or custom styles.
    • Bootstrap tables are not inherited so nested tables can be styled independently of the parent.
    • Use contextual classes to color tables <table>, tables rows <tr> and individual cells <td>.
      • class="table-primary"
      • class="table-secondary"
      • class="table-success"
      • class="table-danger"
      • class="table-warning"
      • class="table-info"
      • class="table-light"
      • class="table-dark"
    • Use .table-striped to add zebra-striping to any table row within the <tbody>.
    • Combine classes for example class="table table-dark table-striped".
    • Add .table-hover to enable a hover state on table rows within a <tbody>.
    • Combine class="table table-dark table-hover" to combine stripes and hoverable rows.
    • Add .table-active class to highlight a table row or cell.
    • Add .table-bordered class for borders on all sides of the table and cells.
    • Add .table-borderless class for a table without borders.
    • Add .table-sm to make any .table more compact by cutting all cell padding in half.
    • Table cells of <thead> are always vertical aligned to the bottom. Table cells in <thead> inherit their alignment from <table> and are aligned to the the top by default.
    • Use the vertical align classes to re-align where needed.
      • vertical-align: middle
      • vertical-align: bottom
    • Border styles, active styles, and table variants are not inherited by nested tables.
    • See how nesting works.
    • Responsive tables allow tables to be scrolled horizontally with ease.
    • Make any table responsive across all viewports by wrapping a .table with .table-responsive for horizontally scrolling tables.
    • Alternatively use .table-responsive{-sm|-md|-lg|-xl|-xxl} to create responsive tables up to a particular breakpoint.
    • Responsive tables use overflow-y: hidden which clips off any content that goes beyond the bottom or top edges of the table.
    • See section for customising in Sass.
    • The <figure> tag can be used for displaying a piece of content such as an image with an optional caption.
    • Use .figure.figure-img and .figure-caption classes to provide some baseline styles for the HTML5 <figure> and <figcaption> elements.
