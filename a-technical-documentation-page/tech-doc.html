Introduction

Bootstrap4 is the most popular HTML, CSS, and JavaScript framework for developing responsive, mobile-first websites.

Bootstrap is a free front-end framework for faster and easier web development.

  • Bootstrap requires the use of the HTML5 doctype.
  • Bootstrap is developed mobile-first. The code is optimised for mobile devices first and components are scaled up as necessary using CSS media queries.
  • Add the responsive viewport meta tag to you <head> to ensure proper rendering and touch zooming for all devices.
  • Bootstrap supports the latest, stable releases of all major browsers and platforms.
  • Bootstrap supports the latest versions of each major platform’s default browsers.
  • You can download Bootstrap or use a CDN. See Getting Started
What you should already know

This guide assumes you have the following basic background:

  • A general understanding of the Internet and the World Wide Web (WWW).
  • Good working knowledge of HyperText Markup Language (HTML).
  • Some programming experience. If you are new to programming, try one of the tutorials linked on the main page about JavaScript.
Layout

This section outlines the components and options for laying out a Bootstrap project including:
wrapping containers, a powerful grid system, flexible media object and responsive utility classes.

Containers are the most basic layout element in Bootstrap and are required when using it's default grid system. Use containers to contain, pad and center the content within them. Containers can be nested but this is not usually required.

    The 3 Bootstrap4 containers:
  • The default .container is a responsive, fixed-width container. This means that it's max-width changes at each responsive breakpoint
  • The .container-fluid class uses width: 100% at all breakpoints so that the content spans the entire width of the viewport
  • Responsive containers using.container-{breakpoint} allows you to specify width: 100% until the specified breakpoint is reached.
      • The following examples are provided at getbootstrap.com
    • <div class="container-sm">100% wide until small breakpoint is reached where it will scale up with md, log and xl</div>
    • <div class="container-md">100% wide until medium breakpoint</div>
    • <div class="container-lg">100% wide until large breakpoint</div>
    • <div class="container-xl">100% wide until extra large breakpoint</div>
  • Layout overview for examples and how to use.

Bootstrap is developed to be mobile-first. It uses a few media queries to create sensible breakpoints for its layouts and interfaces, the breakpoints are mostly based on the minimum viewport widths. They allow for elements to be scaled up as the viewport changes. Refer to the documentation for more information on this and how it works.

Some Bootstrap components use z-index which is a CSS property that helps control layout by providing a third axis to arrange content. We utilize a default z-index scale in Bootstrap that’s been designed to properly layer navigation, tooltips and popovers, modals, and more. Bootstrap uses a default z-index scale that allows proper layer navigation, tooltips and popovers, modals and more. See here for more details. These values start at an arbitrary number that is high and specific enough to ideally avoid conflicts. It is not recommended to customise these values.

  • $zindex-dropdown:1000 !default;
  • $zindex-sticky:1030 !default;
  • $zindex-fixed::1050 !default;
  • $zindex-modal-backdrop:1040 !default;
  • $zindex-modal:1050 !default;
  • $zindex-popover::1060 !default;
  • $zindex-tooltip:1070 !default;

Low single digit numbers such as 1, 2 and 3 overlapping borders and components such as buttons and inputs in input groups, are used to handle for default, hover and active states. On hover/focus/active, a particular element is brought to the forefront with a higher z-index value to show their border over the sibling elements.

Grid

See Grid system which uses a mobile-first flexbox grid to built all sorts of layouts.

The bootstrap grid system is built with Flexbox and uses a series of containers, rows and layouts to layout and align content. It is fully responsive.

  • Containers are a way to centre and horizontally pad your content on the page.
    • .container for a responsive pixel width
    • .container-fluid for width:100% across all viewport and device sizes
    • Rows are a wrapper for columns.
    • Content must be placed in columns in the grid layout
    • Only columns can be direct children of rows.
    • Each column has a horizontal padding or gutter to control the space between columns. The margin can be removed from rows and the padding from columns using .no-gutters on the .row
    • Without specifying a width, each column will have equal width
    • Column classes indicate the number of columns to use out of a possible 12 per row.
    • Column width is set in percentages to they are fluid and sized relative to their parent element.
  • There are five breakpoints (one for each responsive breakpoint) to make the grid responsive.
  • All breakpoints (extra small), small, medium, large and extra large.
  • Grid breakpoints are based on minimum width media queries. This means that they apply to that one breakpoint and all those above it. For example .col-md-4 would apply to medium, large and extra larget devices but not small and extra small breakpoints.
  • Predefined grid classes such as .col-4 can also be used.
  • Bootstrap uses pxs for grid breakpoints and container widths. This is because the viewport width is in pixels and doesn't change with the font size.(Bootstrap does use ems and rems for defining most sizes).
  • See grid options for more details on how the grid system works across multiple
  • breakpoint specific column classes can be used for easy column sizing without an explicit numbered class. For example .col-sm-6
  • Equal width will be applied to columns within a row if no units are supplied.
  • Use .w-100 to create equal-width columns that span multiple-lines. Insert the .w-100 where you want the columns to break to a new line.
  • You can set one column's width and have the other sibling columns in that row automatically resize around it.
  • To size columns based on the natural width of their content use col-{breakpoint}-auto classes.
  • Use .col and .col* classes for grids that are the same for all devices from smallest to largest.
  • Specify a numbered class if you need a particularly sized column. For example class="col-4
  • Change from a stacked grid system to one that becomes horizontal at the smallest breakpoint(sm using a single set of .col-sm-* classes.
  • Use a combination of different classes for each tier if you don't want columns to stack in some tiers.
  • Gutters can be responsively adjusted by breakpoint-specific padding and negative margin utility classes. See gutters for example in the docs.
  • Remove gutters between columns in predefined classes using .no-gutters
  • Use .row-cols-* to quickly set the number of columns that best render the content and layout. This is set in the parent .row as a shortcut. (Normal .col-* apply to individual columns.)
  • Alternatively use the Sass mixin row-cols().
  • Flexbox utilities are used to vertically and horizontally align columns.
      • Vertical Alignment:
    • align-items-start, align-items-center, align-items-end
    • align-self-start, align-self-center, align-self-end
      Horizontal Alignment>
    • row justify-content-start, row justify-content-center, row justify-content-around, row justify-content-between
  • Column wrapping occurs when more than 12 columns are placed on a single row. Each group of extra columns wraps as a unit to the next line.
  • Use width:100% to wrap columns to a new line
  • Reordering: Use .order- classes to control the visual order of the content. These classes are responsive so the order can be set by breakpoint. See reordering for examples.
  • Columns can be moved to the right using .offset-me-* classes to increase the left margin by * columns.
  • .offset-md-4 moves .col-md-4 over 4 columns.
  • Margin utilities such as .mr-auto can be used to force sibling columns away from each other.
  • To nest content within the default grid, add a new .row and a set of .col-sm-* columns within an existing .com-sm-* column.
  • Nested rows should include a set of rows that add up to 12 or fewer.
  • See Sass mixins section on using Sass variables and minins to create custom, semantic and responsive page layouts.

A Technical Documentation page

Following the FreeCodeCamp responsive web design curriculum.
I will document some Bootstrap and JQuery.

  • User Story #1: I can see a main element with a corresponding id="main-doc", which contains the page's main content (technical documentation).
  • User Story #2: Within the #main-doc element, I can see several section elements, each with a class of main-section. There should be a minimum of 5.
  • User Story #3: The first element within each .main-section should be a header element which contains text that describes the topic of that section.
  • User Story #4: Each section element with the class of main-section should also have an id that corresponds with the text of each header contained within it. Any spaces should be replaced with underscores (e.g. The section that contains the header "JavaScript and Java" should have a corresponding id="JavaScript_and_Java").
  • User Story #5: The .main-section elements should contain at least 10 p elements total (not each).
  • User Story #6: The .main-section elements should contain at least 5 code elements total (not each).
  • User Story #7: The .main-section elements should contain at least 5 li items total (not each).
  • User Story #8: I can see a nav element with a corresponding id="navbar".
  • User Story #9: The navbar element should contain one header element which contains text that describes the topic of the technical documentation.
  • User Story #10: Additionally, the navbar should contain link (a) elements with the class of nav-link. There should be one for every element with the class main-section.
  • User Story #11: The header element in the navbar must come before any link (a) elements in the navbar.
  • User Story #12: Each element with the class of nav-link should contain text that corresponds to the header text within each section (e.g. if you have a "Hello world" section/header, your navbar should have an element which contains the text "Hello world").
  • User Story #13: When I click on a navbar element, the page should navigate to the corresponding section of the main-doc element (e.g. If I click on a nav-link element that contains the text "Hello world", the page navigates to a section element that has that id and contains the corresponding header.
  • User Story #14: On regular sized devices (laptops, desktops), the element with id="navbar" should be shown on the left side of the screen and should always be visible to the user.
  • User Story #15: My Technical Documentation page should use at least one media query.
